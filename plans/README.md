# plans/

Piani di lavoro per fase/milestone. Un file per blocco di lavoro coerente.

## Convenzioni
- Naming: `phase<N>-milestone<M>-<slug>.md` (es. `phase1-milestone1-cloud-login.md`).
- Ogni piano contiene: **obiettivo**, **criteri di successo**, **task list**, **rischi/assunzioni**, **fuori scope**.
- Quando un piano è chiuso, marcalo `## Status: DONE <data>` in cima; non cancellare il file (storico utile).
- Se un piano viene riscritto a metà esecuzione, conservare la versione precedente come `*.archived.md`.
