## Decisioni

1. **`_prop()` helper in fan.py**: legge `MF10_PROPERTY_MAP` per costruire
   il payload `{siid, piid, value}` — niente siid/piid hardcodati nel corpo
   dei metodi. Cambiamenti alla property map si propagano automaticamente.
2. **Batch `set_properties` in `turn_on`**: power + speed e/o mode inviati
   in una sola chiamata API quando entrambi forniti. Riduce latenza percepita.
3. **`set_percentage(0)` → `turn_off()`**: `percentage_to_ranged_value`
   con percentage=0 darebbe speed=0 (fuori range); delegare a `turn_off`
   è più corretto e coerente con il comportamento standard HA.
4. **OFF_BEHAVIOR_SOFT deferred**: `turn_off` usa sempre `power=2` (off reale).
   Soft-off (sleep speed 1) rimandato a M4 per evitare deep standby
   non recuperabile — TODO già citato nel CLAUDE.md.
